📋 Pasted Words — What This Means
Detected content insertions larger than expected typing bursts

How it's detected

ProcessAuth monitors your system clipboard every second. When clipboard content exceeds 150 characters, it is classified as a significant paste event. The number of Pasted Words is estimated by dividing total pasted characters by 5 (average word length).

What triggers this count

  • Copying text from a website, PDF, or another document and pasting into the monitored file
  • Pasting content from a previously copied source while the session is active
  • Large block insertions that arrive without corresponding keystroke activity

Score impact

  • 150–300 chars paste → −5 pts
  • 300–600 chars paste → −10 pts
  • > 600 chars paste → −18 pts
  • Rapid back-to-back pastes → additional −5 pts
⚡
A high pasted-word count is a strong signal of externally sourced content. Authentic writers rarely paste large blocks — they type their own ideas progressively.
🚨 Suspicious Events — What This Means
Events flagged by the Hermit Behavioral Engine as anomalous

What counts as a suspicious event?

Each time the behavioral engine detects an anomaly — a large paste, a clipboard correlation, a sudden block insertion, or a confirmed external web source match — it logs one Suspicious Event and applies a proportional score penalty.

Categories of suspicious events

  • Large clipboard paste — content longer than 150 chars copied from outside the document
  • Rapid paste burst — two or more large pastes within 3 seconds of each other
  • Uncorrelated block insertion — large text appears in the document file without any clipboard event
  • External web match — pasted text was found verbatim (or near-verbatim) on DuckDuckGo or Wikipedia
